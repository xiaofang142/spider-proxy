# Spider Proxy v3.0 发布说明

**版本号**: v3.0.0
**发布日期**: 2026-03-26
**开发周期**: 8 周 (2026-03-27 ~ 2026-05-22)

---

## 一、版本概述

v3.0 版本在 v2.0 的基础上实现了专业级调试功能，提供 4 个核心新模块：

1. **请求体替换 (Map Local/Remote)** - 本地文件/远程 URL 替换请求响应体
2. **WebSocket 抓包** - 完整的 WebSocket 连接捕获和消息查看
3. **断点调试 UI 完善** - 手动修改请求/响应内容的编辑器
4. **完整 JavaScript 引擎** - 基于 dart_eval 的沙盒脚本执行

---

## 二、新功能详解

### 2.1 请求体替换模块 (Map Local/Remote)

**文件位置**: `lib/core/mapper/`, `lib/ui/pages/map_local_page.dart`

**功能特性**:
- 支持 Map Local（本地文件替换）和 Map Remote（远程 URL 替换）
- 灵活的匹配规则（URL 模式、域名、HTTP 方法、状态码范围）
- 5 秒 TTL 缓存，支持文件热更新
- 请求体和响应体双向替换

**API 示例**:
```dart
// 创建 Map Local 规则
final rule = BodyMapperRule(
  type: BodyMapType.local,
  target: MapTarget.response,
  localPath: '/path/to/mock.json',
  condition: MatchCondition.urlContains('/api/users'),
);

// 应用规则
final modifiedBody = await bodyMapper.mapRequestBody(url, method, originalBody);
```

---

### 2.2 WebSocket 抓包模块

**文件位置**: `lib/core/proxy/websocket_proxy.dart`, `lib/ui/pages/websocket_page.dart`

**功能特性**:
- 自动捕获 WebSocket 连接
- 实时消息流显示（支持文本和二进制消息）
- 连接状态监控（connecting/open/closing/closed）
- 消息方向标识（发送/接收）
- 连接时长统计

**API 示例**:
```dart
// 创建 WebSocket 连接
final ws = await websocketProxy.createConnection(
  'wss://example.com/socket',
  headers: {'Authorization': 'Bearer token'},
);

// 监听消息
ws.messages.addListener(() {
  final latestMessage = ws.messages.last;
  print('${latestMessage.directionText}: ${latestMessage.payloadString}');
});
```

---

### 2.3 断点调试 UI 完善

**文件位置**: `lib/ui/dialogs/breakpoint_editor_dialog.dart`

**功能特性**:
- Tab 标签页编辑器（URL/Method、Header、Body）
- 请求和响应双向编辑
- Header 编辑器（每行一个，格式：Key: Value）
- Body 编辑器（等宽字体，支持 JSON）
- 修改状态指示器
- "继续"和"修改后继续"双模式

**使用流程**:
1. 在断点列表中点击暂停的请求/响应
2. 点击"修改"按钮打开编辑器
3. 在三个标签页间切换编辑内容
4. 选择"继续"（不修改）或"修改后发送/返回"

---

### 2.4 完整 JavaScript 引擎 (v3.0)

**文件位置**: `lib/core/script/script_engine_v2.dart`

**功能特性**:
- 基于 dart_eval 实现沙盒执行
- 完整的 JavaScript 语法支持
- proxy://context 内置库（ScriptContext API）
- proxy://utils 工具库（print、jsonDecode、jsonEncode）
- 8 个内置脚本模板
- 脚本导入/导出、启用/禁用、批量管理

**脚本示例**:
```javascript
import 'proxy://context' as context;
import 'proxy://utils' as utils;

void onRequest(ctx) {
  ctx.setHeader('X-Custom-Header', 'SpiderProxy-v3');
  ctx.removeHeader('Cookie');

  if (ctx.url.contains('ads.example.com')) {
    ctx.abort();
  }
}

void onResponse(ctx) {
  ctx.setHeader('Access-Control-Allow-Origin', '*');
  utils.print('Response: ${ctx.statusCode}');
}
```

**ScriptContext API**:
| 方法 | 说明 |
|------|------|
| `ctx.url` | 获取/设置请求 URL |
| `ctx.method` | 获取/设置 HTTP 方法 |
| `ctx.headers` | 获取请求头 Map |
| `ctx.body` | 获取/设置请求体 |
| `ctx.statusCode` | 获取/设置状态码 |
| `ctx.setHeader(name, value)` | 设置 Header |
| `ctx.removeHeader(name)` | 删除 Header |
| `ctx.setBody(body)` | 设置响应体 |
| `ctx.setUrl(url)` | 设置 URL |
| `ctx.setMethod(method)` | 设置方法 |
| `ctx.abort()` | 中止请求/响应 |

## 三、升级指南

### 3.1 从 v2.0 升级

**依赖更新**:
```yaml
dependencies:
  # 新增
  dart_eval: ^0.4.0
  flutter_highlight: ^0.7.0
```

**脚本迁移**:
- v2.0 脚本使用简化的 `ScriptEngine`，v3.0 使用 `ScriptEngineV2`
- v3.0 脚本需要导入 `proxy://context` 和 `proxy://utils`
- v2.0 脚本语法（`// @modify-header` 注释）不再支持

**API 变更**:
| v2.0 | v3.0 |
|------|------|
| `Script` | `ScriptV2` |
| `ScriptEngine` | `ScriptEngineV2` |
| `BuiltInScripts` | `BuiltInScriptsV2` |
| `ScriptContext` (无 dart_eval) | `ScriptContext` (dart_eval Bridge) |

### 4.1 性能指标

| 指标 | 目标值 | 实测值 |
|------|-------|-------|
| Map Local 延迟 | < 10ms | ~5ms |

## 五、已知问题

### 5.1 P2 级别

1. **ScriptEngineV2**: `jsonDecode` 和 `jsonEncode` 功能为占位实现，需在后续版本完善
2. **BodyCache**: 文件热更新需手动刷新，自动检测待实现
3. **WebSocketProxy**: 不支持 WebSocket 压缩扩展 (permessage-deflate)

### 5.2 计划中

- v3.1: 完善 JSON 解析/序列化工具
- v3.2: 添加脚本调试功能（断点、单步执行）
