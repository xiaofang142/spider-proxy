## 二、核心功能增强

### 2.1 流量分析模块

#### 2.1.1 实时流量统计图表

**功能描述:**
- 实时显示抓包过程中的流量统计信息
- 支持多种图表类型：折线图、柱状图、饼图

**详细设计:**

| 图表类型 | 展示内容 | 更新频率 |
|---------|---------|---------|
| 请求量趋势 | 每秒/分钟请求数折线图 | 实时 (1s) |
| 响应状态码分布 | 2xx/3xx/4xx/5xx 饼图 | 每 5 秒 |
| 域名请求排行 | Top 10 域名柱状图 | 每 10 秒 |
| 流量大小趋势 | 上行/下行流量折线图 | 实时 (1s) |

**技术实现:**
```kotlin
data class TrafficStats(
    val timestamp: Long,
    val requestCount: Int,
    val responseSize: Long,
    val requestSize: Long,
    val domainStats: Map<String, Int>,
    val statusCodeStats: Map<Int, Int>
)

// 使用 MPAndroidChart 库进行图表渲染
implementation("com.github.PhilJay:MPAndroidChart:v3.1.0")
```

**UI 组件:**
- 可折叠的统计面板
- 图表支持缩放、拖动
- 支持导出图表为 PNG

#### 2.1.2 请求/响应时间线

**功能描述:**
- 可视化展示每个请求的完整生命周期
- 支持瀑布图 (Waterfall) 展示

**时间线阶段:**
```
┌─────────────────────────────────────────────────────────┐
│ DNS 解析  │ TCP 连接 │ SSL 握手 │ 发送请求 │ 等待响应 │ 接收数据 │
│  15ms    │  25ms    │  45ms    │  10ms    │  100ms   │  200ms   │
└─────────────────────────────────────────────────────────┘
```

**详细设计:**

| 阶段 | 说明 | 可优化点 |
|-----|------|---------|
| DNS 解析 | 域名解析耗时 | DNS 缓存 |
| TCP 连接 | 三次握手耗时 | 连接复用 |
| SSL 握手 | TLS 协商耗时 | Session 复用 |
| 发送请求 | 请求体上传耗时 | 压缩 |
| 等待响应 | TTFB (首字节时间) | 服务器优化 |
| 接收数据 | 响应体下载耗时 | 压缩、CDN |

**技术实现:**
```kotlin
data class RequestTimeline(
    val dnsStart: Long,
    val dnsEnd: Long,
    val connectStart: Long,
    val connectEnd: Long,
    val sslStart: Long,
    val sslEnd: Long,
    val requestStart: Long,
    val requestEnd: Long,
    val responseStart: Long,
    val responseEnd: Long
) {
    fun getDnsDuration() = dnsEnd - dnsStart
    fun getConnectDuration() = connectEnd - connectStart
    // ...
}
```

#### 2.1.4 流量趋势分析

**功能描述:**
- 分析长时间抓包的流量趋势
- 识别流量峰值和异常

**分析维度:**
- 时间维度：分钟/小时/天
- 协议维度：HTTP/HTTPS/WebSocket
- 类型维度：API/图片/视频/文档

**异常检测:**
- 请求量突增 (> 200% 基线)
- 响应时间异常 (> 3σ)
- 错误率异常 (4xx/5xx > 10%)

### 2.2 过滤系统

#### 2.2.1 多条件组合过滤

**功能描述:**
- 支持多个过滤条件组合 (AND/OR)
- 实时过滤流量列表

**过滤条件:**

| 条件类型 | 操作符 | 示例 |
|---------|--------|------|
| 域名 | 包含/等于/正则 | `api.example.com` |
| 方法 | 等于 | `GET`, `POST` |
| 状态码 | 范围/等于 | `200-299`, `404` |
| 内容类型 | 包含 | `application/json` |
| 请求大小 | 范围 | `> 1KB`, `< 100KB` |
| 响应时间 | 范围 | `> 500ms` |
| IP 地址 | 等于/段 | `192.168.1.0/24` |

**UI 设计:**
```
┌─────────────────────────────────────────┐
│  过滤器                                  │
│  ┌─────────────────────────────────┐    │
│  │ 域名  [包含 ▼] [api.example.com] │    │
│  │ 方法  [等于 ▼] [POST      ▼]    │    │
│  │ 状态码 [范围 ▼] [200-299]       │    │
│  │                                  │    │
│  │ 逻辑关系：● AND  ○ OR           │    │
│  └─────────────────────────────────┘    │
│  [保存预设] [应用] [重置]                │
└─────────────────────────────────────────┘
```

#### 2.2.2 正则表达式过滤

**功能描述:**
- 支持使用正则表达式进行高级过滤
- 内置正则测试工具

**支持的正则:**
- URL 匹配：`^https?://api\.example\.com/.*`
- 内容匹配：`"error":\s*".*"`
- Header 匹配：`Authorization:\s*Bearer\s*.*`

**正则测试工具:**
```
┌─────────────────────────────────────────┐
│  正则测试                               │
│  正则：^https?://api\.(.*)\.com         │
│  测试：https://api.example.com/users    │

│                                         │
│  常用正则：                             │
│  • 匹配所有 API: .*api.*                │
│  • 匹配图片：.*\.(png|jpg|gif)          │
│  • 匹配错误：[45]\d{2}                  │
└─────────────────────────────────────────┘
```

#### 2.2.3 过滤预设管理
