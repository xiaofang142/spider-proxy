# Spider Proxy 双平台技术架构

## 1. 跨平台方案对比分析

| 方案 | Android | iOS | 代码复用率 | 性能 | 推荐度 |
|------|---------|-----|-----------|------|--------|
| **Flutter** | ✅ | ✅ | 90% UI + 业务逻辑 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **KMP + SwiftUI** | ✅ | ✅ | 70% 业务逻辑 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **纯原生** | Kotlin | Swift | 30% 工具链 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **React Native** | ✅ | ✅ | 85% | ⭐⭐⭐ | ⭐⭐⭐ |

## 2. 推荐架构：Flutter + 原生插件

### 架构图

```
┌──────────────────────────────────────────┐
│          Flutter (共享层 90%)             │
│  - UI 界面 / 状态管理 / 业务逻辑          │
│  - 数据存储 / 网络解析 / 证书管理         │
└──────────────────────────────────────────┘
              ↕ Platform Channel
┌─────────────────┬────────────────────────┐
│  Android 原生    │     iOS 原生           │
│  - VPNService   │  - NetworkExtension    │
│  - 证书安装      │  - 证书安装            │
│  - 后台服务      │  - 后台模式            │
└─────────────────┴────────────────────────┘
```

## 3. 技术选型理由

### 3.1 为什么选择 Flutter

1. **高代码复用率**：90% 的代码可以在双平台共享
2. **优秀性能**：接近原生性能，60fps 流畅体验
3. **成熟生态**：丰富的插件和活跃的社区
4. **开发效率**：热重载功能加速开发迭代
5. **一致体验**：双平台 UI 完全一致

### 3.2 原生插件必要性

| 功能 | Android | iOS |
|------|---------|-----|
| VPN 服务 | VPNService | NetworkExtension |
| 证书安装 | KeyChain | Keychain |
| 后台保活 | Foreground Service | Background Modes |
| 网络拦截 | VpnService | Packet Tunnel |

## 5. 关键技术点

### 5.1 HTTPS 证书管理

- **生成**：使用 Bouncy Castle 生成 CA 证书
- **安装**：双平台分别调用系统 API
- **信任**：引导用户手动信任证书

### 5.2 流量拦截

- **Android**：VpnService 建立虚拟网卡
- **iOS**：NetworkExtension 包隧道
- **代理**：Ktor 实现 HTTP/HTTPS 代理

### 5.3 数据存储

- **本地**：Hive/Isar 存储请求数据
- **导出**：HAR/PCAP 格式支持
- **同步**：可选云同步功能
