# 审核风险评估

> 文档版本：v1.0  
> 创建日期：2026-03-24  
> 适用产品：Spider Proxy

---

## ⚠️ 风险等级说明

| 等级 | 说明 | 应对策略 |
|------|------|---------|
| 🔴 **高风险** | 极可能导致拒审/下架 | 必须解决，否则不上架 |
| 🟡 **中风险** | 可能要求补充材料/修改 | 提前准备应对方案 |
| 🟢 **低风险** | 一般不影响上架 | 注意优化即可 |

---

## 🔴 高风险项

### 1. 抓包工具定位敏感

**风险描述：**
- 抓包工具可能被审核人员视为"黑客工具"
- 可能被认定为用于破解、窃取他人数据
- 同类应用（HttpCanary）曾因此下架

**触发场景：**
- 应用名称含"破解"、"解密"、"攻击"等敏感词
- 功能描述提及"绕过 SSL"、"突破限制"等
- 截图展示破解第三方应用的界面
- 用户举报用于非法用途

**应对策略：**

✅ **重新定位：**
- 官方定位："开发者网络调试工具"
- 副标题："HTTP/HTTPS 协议分析与 API 调试助手"
- 避免使用"抓包"作为主标题（可用"网络分析"替代）

✅ **功能描述优化：**
```
❌ 错误描述：
"轻松抓取任何 APP 的数据包，破解加密协议"

✅ 正确描述：
"帮助开发者分析自己开发的应用的 HTTP/HTTPS 网络请求，
  调试 API 接口，优化网络性能"
```

✅ **截图审查：**
- 仅展示调试自己应用的场景
- 不展示知名第三方应用的抓包界面
- 添加"仅供开发测试使用"水印

✅ **技术文档：**
- 提供详细的技术说明文档
- 强调工具的合法使用场景
- 列出典型开发者使用案例

---

### 2. HTTPS 解密功能

**风险描述：**
- HTTPS 解密涉及证书安装
- 可能被认定为"中间人攻击"工具
- 安全审核会重点关注

**触发场景：**
- 应用要求用户安装根证书
- 解密任意域名的 HTTPS 流量
- 未明确说明证书安全风险

**应对策略：**

✅ **技术实现优化：**
- 采用系统代理模式（无需安装根证书）
- 仅支持用户手动配置代理的应用
- 明确告知用户解密范围

✅ **用户告知：**
```
⚠️ 安全提示：
本功能仅用于开发者调试自己控制的服务端。
请勿用于解密他人应用或网站的 HTTPS 流量，
这可能违反相关法律法规。
```

✅ **证书管理：**
- 提供证书一键卸载功能
- 证书有效期限制（如 24 小时自动过期）
- 证书使用日志记录

✅ **替代方案：**
- 优先支持 HTTP 调试（无风险）
- HTTPS 解密作为"高级功能"，需用户主动开启
- 提供"安全模式"（不解密，仅显示元数据）

---

### 3. 证书安装要求

**风险描述：**
- 要求用户安装描述文件/证书
- iOS 平台对此极为敏感
- 可能触发系统级安全警告

**触发场景：**
- 应用内引导安装证书
- 提供证书下载链接
- 未说明证书用途和风险

**应对策略：**

✅ **iOS 特别处理：**
- 不在应用内直接引导证书安装
- 提供官网教程（应用外）
- 使用系统"描述文件"方式（需用户手动确认）

✅ **明确告知：**
```
证书安装说明：
1. 本证书仅用于调试您自己开发的应用
2. 证书不会上传任何数据到我们的服务器
3. 您可随时在 设置 > 通用 > 描述文件 中删除
4. 如非开发者，请勿安装此证书
```

✅ **权限最小化：**
- 仅申请必要的网络权限
- 不申请通讯录、短信、通话等敏感权限
- 权限用途在隐私政策中明确说明

---

### 4. 网络权限滥用

**风险描述：**
- 申请过多网络相关权限
- 权限用途说明不清
- 可能后台窃取用户数据

**触发场景：**
- 申请"完全网络访问"但无合理说明
- 后台持续网络连接
- 访问用户设备上的其他应用数据

**应对策略：**

✅ **权限清单：**
| 权限 | 必要性 | 用途说明 |
|------|-------|---------|
| 网络访问 | ✅ 必需 | 代理服务器通信 |
| 通知 | ⚠️ 可选 | 抓包完成提醒 |
| 存储 | ⚠️ 可选 | 保存抓包文件 |
| 后台运行 | ⚠️ 可选 | 持续抓包需求 |

✅ **权限说明文案：**
```
网络访问权限：
用于创建本地 VPN 代理，捕获您授权应用的网络请求。
所有数据仅在本地处理，不会上传到任何服务器。

存储权限：
用于保存您主动导出的抓包文件（HAR/JSON 格式）。
不会访问您设备上的其他文件。
```

✅ **权限动态申请：**
- 首次启动仅申请必要权限
- 高级功能使用时再申请对应权限
- 提供"拒绝后引导"（说明为什么需要）

## 🟡 中风险项

### 5. 隐私政策不合规

**风险描述：**
- 隐私政策内容不完整
- 未说明数据收集范围
- 未提供用户权利行使方式

**应对策略：**
- 使用标准模板（阿里云/腾讯云生成器）
- 明确列出收集的数据类型和用途
- 提供数据删除/导出功能
- 设置隐私政策弹窗（首次启动）

---

### 6. 用户协议缺失免责条款

**风险描述：**
- 用户滥用工具导致法律问题
- 开发者可能承担连带责任

**应对策略：**
```
免责声明：
1. 本工具仅供学习和开发测试使用
2. 用户不得用于破解、窃取他人数据等非法用途
3. 因用户违法使用导致的后果由用户自行承担
4. 开发者保留对违法用户终止服务的权利
```

---

### 7. 应用名称/图标侵权

**风险描述：**
- 名称与知名应用相似
- 图标设计侵犯他人商标

**应对策略：**
- 名称：Spider Proxy（自有品牌）
- 图标：原创设计（蜘蛛 + 网络元素）
- 避免使用"微信助手"、"淘宝破解"等蹭热度命名

---

### 8. 第三方组件声明

**风险描述：**
- 使用开源组件未声明
- 违反开源许可证要求

**应对策略：**
- 在"关于"页面列出所有第三方组件
- 遵守各组件的许可证要求（MIT/Apache/GPL 等）
- 提供开源组件源码获取方式

---

## 🟢 低风险项

### 9. 界面美观度

**风险描述：**
- 界面粗糙影响用户体验
- 可能被用户差评

**应对策略：**
- 参考主流工具的设计风格
- 保持界面简洁清晰
- 提供深色模式

---

### 10. 功能稳定性

**风险描述：**
- 抓包失败率高
- 应用崩溃

**应对策略：**
- 充分测试各品牌手机兼容性
- 提供崩溃日志上报（需用户同意）
- 设置用户反馈渠道

## 💡 各平台审核特点

### 华为应用市场
- **特点：** 最严格、流程规范
- **常见拒因：** 资质不全、隐私政策不合规
- **建议：** 材料准备齐全再提交

### 小米应用商店
- **特点：** 中等严格、响应较快
- **常见拒因：** 功能描述敏感
- **建议：** 强调"开发者工具"定位

### OPPO/VIVO
- **特点：** 中等严格、下沉市场
- **常见拒因：** 界面体验差、权限过多
- **建议：** 简化界面、权限最小化

### 应用宝（腾讯）
- **特点：** 严格、竞品审查
- **常见拒因：** 与腾讯产品冲突、安全疑虑
- **建议：** 避免与腾讯电脑管家等功能重叠

### App Store
- **特点：** 严格但透明、申诉渠道畅通
- **常见拒因：** 隐私政策、功能描述不清
- **建议：** 详细回复审核人员问题
