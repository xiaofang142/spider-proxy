## 一、环境准备

### 1.1 系统要求

| 系统 | 最低要求 | 推荐配置 |
|-----|---------|---------|
| macOS | 12+ | 13+ |
| Windows | 10+ | 11+ |
| Linux | Ubuntu 20.04+ | Ubuntu 22.04+ |

### 1.2 安装 Flutter

**macOS:**
```bash
# 使用 Homebrew
brew install --cask flutter

# 或手动安装
cd ~
git clone https://github.com/flutter/flutter.git -b stable
export PATH="$PATH:$HOME/flutter/bin"
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.zshrc
source ~/.zshrc
```

**Windows:**
```powershell
# 下载并解压 Flutter SDK
# https://storage.googleapis.com/flutter_infra_release/releases/stable/windows/flutter_windows_3.24.0-stable.zip

# 添加环境变量
setx PATH "%PATH%;C:\src\flutter\bin"
```

**Linux:**
```bash
sudo apt-get install -y curl git unzip xz-utils zip
cd ~
git clone https://github.com/flutter/flutter.git -b stable
export PATH="$PATH:$HOME/flutter/bin"
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.bashrc
source ~/.bashrc
```

## 二、安装 Android Studio（仅 Android 开发）

### 2.1 安装

**macOS:**
```bash
brew install --cask android-studio
```

**Linux:**
```bash
sudo snap install android-studio --classic
```

**Windows:**
从 https://developer.android.com/studio 下载安装

### 2.2 配置 SDK

1. 打开 Android Studio
2. 进入 `Preferences > Appearance & Behavior > System Settings > Android SDK`
3. 安装:
   - Android SDK Platform (API 24-35)
   - Android SDK Build-Tools
   - Android Emulator
   - Android SDK Platform-Tools

### 2.3 创建模拟器

```bash
# 列出可用镜像
avdmanager list target

# 创建 AVD
avdmanager create avd -n pixel_6 -k "system-images;android-34;google_apis_playstore;arm64-v8a" -d pixel_6

# 启动模拟器
emulator -avd pixel_6
```

## 四、使用指南

### 4.1 启动代理

1. 打开应用
2. 在首页点击「启动代理」按钮
3. 应用会显示「代理运行中」状态

### 4.2 配置设备代理

**方法一：手动配置（推荐开发调试使用）**

1. 打开设备 WiFi 设置
2. 长按当前网络 > 修改网络
3. 启用「代理」> 「手动」
4. 主机名：运行应用的设备 IP（模拟器用 `10.0.2.2`）
5. 端口：`8888`
6. 保存

**方法二：VPN 模式（待实现）**

- 启动代理时会自动创建 VPN 连接
- 无需手动配置

### 4.3 查看抓包

1. 在设备上浏览网页或使用应用
2. 返回 Spider Proxy 应用
3. 切换到「抓包」标签页
4. 查看捕获的网络请求

### 4.4 搜索和过滤

1. 在抓包列表页面，使用顶部搜索框输入关键词
2. 点击过滤图标，可以选择：
   - 请求方法（GET, POST, PUT, DELETE...）
   - 状态码（2xx, 3xx, 4xx, 5xx）

### 4.5 HTTPS 解密

1. 进入「设置」页面
2. 点击「安装 CA 证书」
3. 按照提示安装证书：
   - 打开设备设置
   - 进入「安全」> 「加密和凭据」
   - 选择「从存储安装 CA 证书」
   - 选择 Spider Proxy 证书文件
4. 安装完成后，启用「HTTPS 解密」开关

## 五、常见问题

### 5.1 无法启动代理

**问题:** 点击「启动代理」后报错

**解决方案:**
1. 检查是否授予了网络权限
2. 确保端口 8888 未被占用
3. 查看日志输出：`flutter logs --grep="proxy"`

### 5.2 抓不到包

**问题:** 代理已启动，但抓包列表为空

**解决方案:**
1. 确认设备代理配置正确
2. 检查防火墙设置
3. 尝试访问 HTTP 网站（如 http://example.com）测试
4. 确认应用允许明文流量（`usesCleartextTraffic="true"`）

### 5.3 HTTPS 网站无法解密

**问题:** 能抓到 HTTPS 请求，但内容是加密的

**解决方案:**
1. 确保已安装 CA 证书
2. 证书已信任（在系统证书存储中）
3. 部分应用使用 SSL Pinning，无法解密（正常现象）

### 5.4 模拟器网络问题

**问题:** 使用模拟器时无法连接

**解决方案:**
- Android 模拟器使用 `10.0.2.2` 作为主机地址
- 在应用设置中修改代理主机为 `10.0.2.2`
