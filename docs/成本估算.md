## 💡 成本优化策略

### 1. 模型选择优化

```yaml
# 根据任务复杂度选择模型
modelSelection:
  rules:
    - taskType: "simple-format"
      model: "gpt-4o-mini"  # $0.00015/次
      savings: "94% vs GPT-4o"
    
    - taskType: "security-analysis"
      model: "claude-3.5-sonnet"  # 更准确
      reason: "安全审查需要高精度"
    
    - taskType: "batch-processing"
      model: "qwen-plus"  # ¥0.004/1K
      savings: "90% vs GPT-4o"
    
    - taskType: "sensitive-data"
      model: "ollama/llama3"  # 免费
      reason: "隐私保护 + 零成本"
```

### 2. 批量处理

**策略:** 合并多个请求为一次调用

```
单独处理 (10 个请求):
10 × 800 tokens × $0.0003 = $0.024

批量处理 (1 次调用):
5000 tokens × $0.0003 = $0.0015

节省：93.75%
```

### 3. 缓存复用

**策略:** 相同输入直接返回缓存结果

```
缓存命中率：60%
日调用量：100 次
实际 API 调用：40 次

成本节省：60%
```

### 4. 本地模型分流

**策略:** 简单任务/敏感数据用本地模型

```
任务分布:
- 简单格式化：40% → Ollama (免费)
- 通用分析：40% → GPT-4o-mini
- 复杂任务：20% → GPT-4o/Claude

月度成本:
原方案 (全部云端): $50
优化后：$25
节省：50%
```

### 5. Prompt 优化

**策略:** 精简输入，减少 token 消耗

```
优化前:
- 完整请求 JSON: 1000 tokens
- 冗余说明：200 tokens
总计：1200 tokens

优化后:
- 精简请求：500 tokens
- 精简说明：100 tokens
总计：600 tokens

节省：50%
```
