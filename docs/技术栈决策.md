## 一、决策背景

在项目审计过程中，发现现有文档中存在技术栈描述不一致的问题：
- 部分文档描述使用 Kotlin + Jetpack Compose（Android 原生）
- 实际代码使用 Flutter/Dart
- AI 技术架构文档描述使用 React/Vue

本文档旨在明确并统一技术栈选型。

## 二、最终技术栈决策

### 2.1 核心框架

| 决策项 | 选型 | 理由 |
|-------|------|------|
| **应用框架** | Flutter 3.24+ | 跨平台、高性能、单一代码库 |
| **开发语言** | Dart 3.5+ | Flutter 官方语言、类型安全 |
| **状态管理** | flutter_redux 6.0+ | 已采用，保持连续性 |
| **本地存储** | Hive 2.2+ | 轻量级、高性能、Flutter 友好 |

### 2.2 平台策略

| 平台 | 优先级 | 版本 | 说明 |
|-----|--------|------|------|
| Android | P0 | API 24+ (Android 7.0) | v1.0 MVP 目标平台 |
| iOS | P1 | iOS 14+ | v2.0 扩展平台 |

## 三、技术栈对比分析

### 3.1 框架选型对比

| 标准 | Flutter | Kotlin 原生 | React Native |
|-----|---------|-----------|-------------|
| 跨平台能力 | ✅ 优秀 | ❌ 仅 Android | ✅ 良好 |
| 性能 | ✅ 接近原生 | ✅ 原生 | 🟡 依赖桥接 |
| 开发效率 | ✅ 热重载 | 🟡 编译较慢 | ✅ 热重载 |
| 原生能力 | ✅ 平台通道 | ✅ 原生 | ✅ 原生模块 |
| 社区支持 | ✅ 活跃 | ✅ 稳定 | ✅ 活跃 |
| 学习曲线 | 🟡 中等 | 🟡 需 Kotlin | ✅ Web 开发者友好 |
| **综合评分** | **9/10** | **7/10** | **7.5/10** |

### 3.2 为什么选择 Flutter

**核心优势：**
1. **单一代码库** - Android/iOS共享 90%+ 代码
2. **高性能** - Skia 引擎直接渲染，60/120fps
3. **热重载** - 秒级开发迭代
4. **Material Design** - 内置高质量 UI 组件
5. ** Dart 语言** - 类型安全、空安全、异步友好

**风险评估：**
- 包体积：相比原生增加约 5-10MB（可接受）
- 平台特定功能：需要 Platform Channel（已有成熟方案）

## 四、架构决策记录

### ADR-001: 选择 Flutter 作为应用框架

**状态:** ✅ 采纳
**日期:** 2026-03-25

**上下文:**
项目需要同时支持 Android 和 iOS 平台，需要在开发效率和性能之间取得平衡。

**决策:**
采用 Flutter 3.24+ 作为应用框架。

**后果:**
- ✅ 代码复用率 > 90%
- ✅ 开发效率提升 40%
- ⚠️ 需要学习 Dart 语言
- ⚠️ 包体积增加 5-10MB

---

### ADR-002: 使用 flutter_redux 进行状态管理

**状态:** ✅ 采纳（保持现有）
**日期:** 2026-03-25

**上下文:**
项目已使用 flutter_redux，评估是否继续使用或迁移到其他方案。

**决策:**
继续使用 flutter_redux，未来可考虑迁移到 Riverpod。

**后果:**
- ✅ 保持代码连续性
- ✅ 团队已有相关知识
- ⚠️ 代码相对冗长

---

### ADR-003: 使用 Hive 作为本地存储

**状态:** ✅ 采纳
**日期:** 2026-03-25

**上下文:**
需要轻量级、高性能的本地存储方案。

**决策:**
采用 Hive 2.2+ 作为本地存储方案。

**后果:**
- ✅ 性能优于 SharedPreferences
- ✅ 支持数据类型丰富
- ✅ Flutter 友好
- ⚠️ 需要学习 Hive API

## 五、禁止使用的技术

为避免技术栈混乱，以下技术**禁止**在本项目中未经批准使用：

| 技术 | 原因 | 替代方案 |
|-----|------|---------|
| Kotlin (应用层) | 与跨平台策略冲突 | Platform Channel 原生代码除外 |
| Swift (应用层) | 与跨平台策略冲突 | Platform Channel 原生代码除外 |
| React/Vue | 与选定架构冲突 | Flutter Widgets |
| Redux.js | 平台不匹配 | flutter_redux |
| Room/SQLite (直接) | 已有 Hive 方案 | Hive, Isar |
| Retrofit | Kotlin 库 | Dio (如需 HTTP 客户端) |
