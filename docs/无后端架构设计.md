### 1.2 架构决策

| 决策点 | 原方案 | 新方案 | 理由 |
|--------|--------|--------|------|
| 后端服务 | Ktor 服务器 | **移除** | 零服务器成本，零运维 |
| 用户认证 | 账号系统 | **移除** | 本地应用无需登录 |
| 数据存储 | 云端数据库 | **本地 SQLite** | 数据隐私更好 |
| AI 分析 | 后端代理 | **直接调用 LLM API** | 降低延迟，简化架构 |
| 内购管理 | 自建 | **RevenueCat** | 节省 50% 开发时间 |

## 二、核心模块设计

### 2.1 VPN 服务/代理引擎

**技术选型:**
- **iOS:** NetworkExtension (NEPacketTunnelProvider)
- **Android:** VpnService + OkHttp

**关键功能:**
```
┌─────────────────────────────────────────┐
│           VPN 服务模块                   │
├─────────────────────────────────────────┤
│ • 启动/停止 VPN 隧道                     │
│ • 流量拦截与转发                         │
│ • 证书安装与管理                         │
│ • 后台保活 (Android)                     │
│ • 连接状态监控                           │
└─────────────────────────────────────────┘
```

**代码结构:**
```
lib/
├── vpn/
│   ├── vpn_service.dart        # 跨平台接口
│   ├── vpn_service_ios.dart    # iOS 实现 (MethodChannel)
│   └── vpn_service_android.dart # Android 实现 (MethodChannel)
```

### 2.2 抓包与解析模块

**技术栈:**
- HTTP 解析：`package:http_parser`
- SSL 解密：MITM 证书 + 自定义 TrustManager
- 数据解析：JSON/XML/Protobuf 自动识别

**处理流程:**
```
请求 → 拦截 → SSL 解密 → 解析 → 存储 → UI 展示
  │                                              │
  └────────────────── 响应 ←─────────────────────┘
```

### 2.3 AI 分析模块 (本地直连)

**架构变化:**
```
【原方案】App → 后端 API → LLM API → 后端 → App
【新方案】App → LLM API (直接调用) → App
```

**支持的 LLM 提供商:**
| 提供商 | API 端点 | 成本 | 延迟 |
|--------|---------|------|------|
| OpenAI | api.openai.com | $0.01/1K tokens | ~500ms |
| Claude | api.anthropic.com | $0.003/1K tokens | ~800ms |
| 通义千问 | dashscope.aliyuncs.com | ¥0.002/1K tokens | ~300ms |
| DeepSeek | api.deepseek.com | ¥0.001/1K tokens | ~400ms |

**本地缓存策略:**
```dart
class AICache {
  // 相同请求缓存 24 小时
  Future<String?> get(String requestId);
  Future<void> set(String requestId, String result);
}
```

### 2.4 数据存储模块 (SQLite)

## 三、第三方服务集成

### 3.1 RevenueCat (内购管理)

**为什么选择 RevenueCat:**

| 优势 | 说明 |
|------|------|
| 跨平台 | 一套代码对接 App Store + Google Play |
| 自动验证 | 自动处理收据验证，防破解 |
| 订阅管理 | 自动处理续订/退款/恢复购买 |
| 免费额度 | ≤$2.5k 月收入完全免费 |
| 试用管理 | 内置免费试用配置 |

**集成步骤:**

1. **安装 SDK:**
```yaml
dependencies:
  purchases_flutter: ^6.0.0
```

2. **初始化:**
```dart
await Purchases.configure(
  PurchasesConfiguration('rc_你的_api_key'),
);
```

3. **检查订阅状态:**
```dart
final customerInfo = await Purchases.getCustomerInfo();
final hasActiveSubscription = customerInfo.entitlements.active.isNotEmpty;
```

## 四、无后端优势分析

### 4.1 成本对比

| 成本项 | 有后端方案 | 无后端方案 | 节省 |
|--------|-----------|-----------|------|
| 服务器 | ¥5,000/月 | ¥0 | 100% |
| 运维人力 | ¥35,000/月 | ¥0 | 100% |
| 后端开发 | ¥35,000/月 | ¥0 | 100% |
| 数据库 | ¥1,000/月 | ¥0 | 100% |
| CDN/存储 | ¥2,000/月 | ¥0 | 100% |
| **6 个月总计** | **¥468,000** | **¥0** | **¥468,000** |

### 4.2 开发效率

| 指标 | 有后端 | 无后端 | 提升 |
|------|--------|--------|------|
| 开发周期 | 36 周 | 24 周 | 33% |
| 代码量 | ~50K 行 | ~30K 行 | 40% |
| 测试复杂度 | 高 | 中 | 50% |
| 部署流程 | 复杂 | 简单 | 70% |

### 4.3 数据隐私

```
【有后端方案】
用户数据 → App → 后端服务器 → 数据库
            ↓
        数据泄露风险

【无后端方案】
用户数据 → App → 本地 SQLite
            ↓
        仅用户设备可访问
```

## 五、技术风险与应对

### 5.1 风险清单

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| LLM API 被滥用 | 中 | 高 | 本地限流 + 缓存 |
| 内购破解 | 中 | 高 | RevenueCat 收据验证 |
| 本地数据泄露 | 低 | 中 | 数据加密存储 |
| App 体积过大 | 低 | 低 | 按需加载 + 代码分割 |

### 5.2 安全策略

**数据加密:**
```dart
import 'package:encrypt/encrypt.dart';

class LocalStorage {
  final _key = Key.fromUtf8('32 字符密钥____________');
  final _iv = IV.fromLength(16);
  
  Future<void> saveSensitive(String data) async {
    final encrypter = Encrypter(AES(_key));
    final encrypted = encrypter.encrypt(data, iv: _iv);
    // 存储 encrypted.base64
  }
}
```

**内购验证:**
```dart
// RevenueCat 自动处理收据验证

final isPro = customerInfo.entitlements.active['pro'] != null;
```
