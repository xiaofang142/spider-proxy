## 二、RevenueCat 集成方案

### 2.1 账户配置流程

**步骤 1: 创建 RevenueCat 账户**
```
1. 访问 https://app.revenuecat.com
2. 注册账户 (GitHub/Google/邮箱)
3. 创建新项目 "Spider Proxy"
4. 选择平台：iOS + Android
```

**步骤 2: 配置 iOS**
```
1. 在 RevenueCat 创建 iOS App
2. 填写 Bundle ID: com.spiderproxy.app
3. 上传 In-App Purchase Key (从 App Store Connect)
4. 配置 Shared Secret (用于收据验证)
```

**步骤 3: 配置 Android**
```
1. 在 RevenueCat 创建 Android App
2. 填写 Package Name: com.spiderproxy.app

```

**步骤 4: 创建产品**
```
在 RevenueCat 仪表盘:
1. 创建 Entitlement: "pro"
2. 创建 Offerings: "default"
3. 创建 Products:
   - 7 天免费试用 + ¥9.9 买断 (iOS)
   - 7 天免费试用 + ¥9.9 买断 (Android)
```

### 2.2 Flutter 代码集成

**依赖配置:**
```yaml
# pubspec.yaml
dependencies:
  purchases_flutter: ^6.25.0
```

**初始化代码:**
```dart
// lib/services/purchase_service.dart
import 'package:purchases_flutter/purchases_flutter.dart';

class PurchaseService {
  static const String _apiKey = 'rc_你的_api_key';
  
  Future<void> init() async {
    // 配置 RevenueCat
    await Purchases.configure(
      PurchasesConfiguration(_apiKey),
      appUserID: await _getLocalUserId(),
    );
    
    // 监听用户状态变化
    Purchases.addCustomerInfoUpdateListener(_onCustomerInfoUpdated);
  }
  
  Future<String> _getLocalUserId() async {
    // 生成本地唯一 ID (UUID)
    final uuid = const Uuid().v4();
    // 存储到本地
    await SharedPreferences.getInstance()
      .then((prefs) => prefs.setString('user_id', uuid));
    return uuid;
  }
  
  void _onCustomerInfoUpdated(CustomerInfo customerInfo) {
    // 更新 UI 状态
    final isPro = customerInfo.entitlements.active['pro'] != null;
    // 通知 UI 层
    PurchaseStatusNotifier.instance.update(isPro);
  }
}
```

**检查订阅状态:**
```dart
Future<bool> isProUser() async {

```

**购买流程:**
```dart
Future<PurchaseResult> purchasePro() async {
  try {
    // 获取可用产品
    final offerings = await Purchases.getOfferings();
    final product = offerings.current?.availablePackages.first;
    
    if (product == null) {
      return PurchaseResult.error('产品不可用');
    }
    
    // 发起购买
    final result = await Purchases.purchasePackage(product);
    
    // 检查购买结果
    final customerInfo = result.customerInfo;
    if (customerInfo.entitlements.active['pro'] != null) {
      return PurchaseResult.success;
    } else {
      return PurchaseResult.pending;
    }
  } on PurchasesException catch (e) {
    return PurchaseResult.error(e.message);
  }
}
```

**恢复购买:**
```dart
Future<bool> restorePurchases() async {
  try {
    final customerInfo = await Purchases.restorePurchases();
    return customerInfo.entitlements.active['pro'] != null;
  } catch (e) {
    return false;
  }
}
```

## 三、平台原生 IAP 方案 (备选)

### 3.1 iOS StoreKit 2

**适用场景:** 仅 iOS 版本，不计划上 Android

**代码示例:**
```swift
// iOS 原生代码 (通过 Platform Channel 调用)
import StoreKit

@available(iOS 15.0, *)
class IAPManager: ObservableObject {
    func purchase() async throws {
        guard let product = await products.first(where: { $0.id == "com.spiderproxy.pro" }) else {
            throw IAPError.productNotFound
        }
        
        let result = try await product.purchase()
        
        switch result {
        case .success(let verification):
            let transaction = try verification.payloadValue
            await transaction.finish()
            // 解锁功能
        case .pending:
            // 等待验证
        case .userCancelled:
            // 用户取消
        }
    }
}
```

**缺点:**
- ❌ 仅支持 iOS
- ❌ 需自建收据验证服务器
- ❌ 需处理订阅状态同步

### 3.2 Android Play Billing

**适用场景:** 仅 Android 版本

**代码示例:**
```kotlin
// Android 原生代码
class BillingManager(context: Context) {
    private val billingClient = BillingClient.newBuilder(context)
        .setListener(purchasesUpdatedListener)
        .enablePendingPurchases()
        .build()
    
    fun purchase(activity: Activity) {
        val flowParams = BillingFlowParams.newBuilder()
            .setProductDetailsParamsList(
                listOf(
                    BillingFlowParams.ProductDetailsParams.newBuilder()
                        .setProductDetails(productDetails)
                        .build()
                )
            )
            .build()
        
        billingClient.launchBillingFlow(activity, flowParams)
    }
}
```

**缺点:**
- ❌ 仅支持 Android
- ❌ 审核流程复杂
- ❌ 需处理多种支付场景

### 4.2 试用状态管理

```dart
enum TrialStatus {
  active,      // 试用中
  expired,     // 已过期
  purchased,   // 已购买
  ineligible,  // 不符合试用资格
}

class TrialManager {
  Future<TrialStatus> checkStatus() async {
    final customerInfo = await Purchases.getCustomerInfo();
    
    // 检查是否已购买
    if (customerInfo.entitlements.active['pro'] != null) {
      return TrialStatus.purchased;
    }
    
    // 检查试用状态
    final proEntitlement = customerInfo.entitlements.all['pro'];
    if (proEntitlement != null) {
      if (proEntitlement.isActive) {
        return TrialStatus.active;
      } else {
        return TrialStatus.expired;
      }
    }
    
    // 未开始试用
    return TrialStatus.ineligible;
  }
}
```
