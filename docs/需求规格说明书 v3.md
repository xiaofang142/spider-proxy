## 一、文档概述

### 1.1 目的

本文档定义 Spider Proxy v3.0 的功能需求，在 v2.0 基础上增加专业级调试能力。

### 1.2 范围

- **平台**: Android
- **版本**: v3.0 (专业功能版本)
- **目标**: 提供媲美桌面工具的专业调试能力

### 1.3 与 v2.0 的关系

v3.0 在 v2.0 已完成功能基础上进行增强：
- v2.0 已实现：流量分析增强、请求重写、断点调试、脚本系统
- v3.0 新增：请求体替换、WebSocket 抓包、断点调试 UI 完善、完整 JS 引擎

## 四、核心功能需求详述

### 4.1 请求体替换 (Map Local/Remote)

#### F-301: Map Local (本地文件替换响应体)

**描述**: 将本地文件内容替换远程响应体，用于本地开发调试

**功能要求**:
- 支持文件类型：JSON、XML、HTML、JS、CSS、图片等
- 支持路径匹配规则（精确匹配、正则匹配）
- 支持 Content-Type 自动识别
- 支持热更新（修改文件后无需重启）

**验收标准**:
- [ ] 支持常见文件格式
- [ ] 匹配规则灵活配置
- [ ] 替换后 Content-Length 自动更新
- [ ] 文件修改后自动生效

---

#### F-302: Map Remote (远程响应替换)

**描述**: 将一个远程响应替换为另一个远程响应

**功能要求**:
- 支持 URL 重定向到远程地址
- 支持响应状态码替换
- 支持响应头继承/覆盖选项

**验收标准**:
- [ ] 支持 HTTP/HTTPS 远程地址
- [ ] 响应头处理正确
- [ ] 状态码替换生效

---

#### F-303: 请求体修改

**描述**: 在请求发送前修改请求体内容

**功能要求**:
- 支持 JSON、XML、Form 数据修改
- 支持编辑器高亮显示
- 支持内容验证（JSON 格式检查）

**验收标准**:
- [ ] 支持主流数据格式
- [ ] 编辑器高亮正常
- [ ] 格式验证正确

---

#### F-304: 响应体修改

**描述**: 在响应返回前修改响应体内容

**功能要求**:
- 支持 JSON、XML、HTML 修改
- 支持编辑器高亮显示
- 支持内容验证

**验收标准**:
- [ ] 支持主流数据格式
- [ ] 编辑器高亮正常
- [ ] 格式验证正确

---

### 4.2 WebSocket 抓包

#### F-401: WebSocket 连接捕获

**描述**: 捕获 WebSocket 连接建立过程

**功能要求**:
- 捕获 WebSocket 握手请求
- 记录升级协议过程
- 显示连接状态（连接中/已连接/已关闭）

**验收标准**:
- [ ] 握手请求正常捕获
- [ ] 连接状态正确显示
- [ ] 支持多个并发连接

---

#### F-402: WebSocket 消息查看

**描述**: 查看 WebSocket 收发的消息内容

**功能要求**:
- 区分客户端发送/服务端推送
- 支持文本消息显示
- 支持二进制消息（图片/音频/文件）预览
- 显示消息方向（发送/接收）、时间戳、大小

**验收标准**:
- [ ] 消息内容完整显示
- [ ] 二进制消息可预览
- [ ] 消息方向标识清晰

---

#### F-403: 实时消息流

**描述**: 实时显示 WebSocket 消息流

**功能要求**:
- 消息实时滚动显示
- 支持暂停/继续
- 支持消息过滤（按内容、方向）
- 支持消息搜索

**验收标准**:
- [ ] 消息实时显示
- [ ] 暂停/继续正常
- [ ] 过滤搜索生效

---

#### F-404: WebSocket 连接详情

**描述**: 查看 WebSocket 连接的详细信息

**功能要求**:
- 握手请求/响应头详情
- 连接元数据（协议版本、扩展、子协议）
- 帧信息显示（FIN、Opcode、Mask、Payload length）
- 关闭状态码和原因

**验收标准**:
- [ ] 握手信息完整
- [ ] 帧信息显示正确
- [ ] 关闭原因清晰展示

## 五、非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| Map Local 延迟 | < 10ms | 本地文件替换延迟 |
| Map Remote 延迟 | < 100ms | 远程响应替换延迟 |
| WebSocket 消息延迟 | < 50ms | 消息捕获延迟 |
| 断点响应时间 | < 200ms | UI 打开编辑器时间 |
| 脚本执行时间 | < 100ms | 单脚本执行时间 |

### 5.2 内存要求

- 基础内存占用：< 300MB
- WebSocket 长连接：每个连接 < 10MB
- Map Local 缓存：< 50MB

### 5.3 兼容性要求

- Android 7.0 (API 24) 及以上
- 向下兼容 v2.0 所有功能
- 脚本格式兼容（简化语法 → 完整语法）

## 六、用户故事

### 6.1 Map Local 用户故事

**场景**: 前端开发者需要 Mock 后端接口进行开发

> 作为一个前端开发者
> 我希望将本地 JSON 文件替换远程 API 响应
> 以便在后端接口未完成时进行前端开发

**验收标准**:
1. 创建 Map Local 规则，匹配目标 API URL
2. 选择本地 JSON 文件
3. 启用规则后，API 返回本地文件内容
4. 修改本地文件后，无需重启自动生效

---

### 6.2 WebSocket 用户故事

**场景**: 开发者需要调试实时通信功能

> 作为一个移动开发者
> 我希望查看 WebSocket 的实时消息流
> 以便排查通信问题

**验收标准**:
1. 启动抓包后，WebSocket 连接自动捕获
2. 在列表页可识别 WebSocket 流量
3. 点击进入实时消息流页面
4. 消息按时间顺序显示，支持过滤

---

### 6.3 断点调试用户故事

**场景**: 测试人员需要修改请求参数进行测试

> 作为一个测试工程师
> 我希望在请求发送前手动修改参数
> 以便测试各种边界情况

**验收标准**:
1. 创建请求前断点规则
2. 请求被拦截，显示编辑器
3. 修改请求参数后继续发送
4. 服务器收到修改后的请求

## 七、依赖与约束

### 7.1 技术依赖

| 依赖 | 用途 | 版本 |
|------|------|------|
| dart_eval | JavaScript 引擎 | ^0.4.0+ |
| web_socket_channel | WebSocket 支持 | ^2.4.0+ |

- 完整 JavaScript 引擎可能增加包体积约 5-10MB
