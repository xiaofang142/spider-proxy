#### SEC-001: CA 私钥 KeyStore 存储实现方案

**当前实现（不安全）：**
```dart
// certificate_manager.dart:110-115
await certFile.writeAsString(_caCertificatePem!);
await keyFile.writeAsString(_caPrivateKeyPem!);
// ❌ 私钥以 PEM 明文存储在 filesDir
```

**建议实现（Android KeyStore）：**
```kotlin
// SslInterceptor.kt
class SslInterceptor(private val certsDir: File, private val context: Context) {
    private lateinit var keyStore: KeyStore

    fun initialize() {
        keyStore = KeyStore.getInstance("AndroidKeyStore").apply { load(null) }

        if (!keyStore.containsAlias(KEYSTORE_ALIAS)) {
            generateKeyInKeyStore()
        }
    }

    private fun generateKeyInKeyStore() {
        val keyGenerator = KeyPairGenerator.getInstance(
            KeyProperties.KEY_ALGORITHM_RSA, "AndroidKeyStore"
        )
        keyGenerator.initialize(
            KeyGenParameterSpec.Builder(
                KEYSTORE_ALIAS,
                KeyProperties.PURPOSE_SIGN or KeyProperties.PURPOSE_ENCRYPT or KeyProperties.PURPOSE_DECRYPT
            )
                .setKeySize(2048)
                .setDigests(KeyProperties.DIGEST_SHA256)
                .setEncryptionPaddings("PKCS1v15")
                .setUserAuthenticationRequired(false)  // 无需用户认证
                .setKeyValidityStart(Date())
                .build()
        )
        keyGenerator.generateKeyPair()
    }

    private fun getPrivateKey(): PrivateKey {
        val entry = keyStore.getEntry(KEYSTORE_ALIAS, null) as KeyStore.PrivateKeyEntry
        return entry.privateKey  // ✅ 私钥不可导出
    }
}
```

**KeyStore 安全特性：**
- ✅ 私钥存储在硬件支持的密钥库中（如果设备支持）
- ✅ 即使 root 用户也无法直接提取私钥
- ✅ 应用卸载时自动删除

#### SEC-002: 证书 24 小时过期机制

**当前实现：**
```dart
// certificate_manager.dart:88-89
val expiry = DateTime(now.year + 10, now.month, now.day);
// ❌ 10 年有效期
```

**建议实现：**
```kotlin
// SslInterceptor.kt
companion object {
    private const val HOST_CERT_VALIDITY_MS = 24 * 60 * 60 * 1000L // 24 小时
}

fun getSSLContextForHost(host: String): SSLContext? {
    val cached = certCache[host]
    if (cached != null && !cached.isExpired) {
        return cached.sslContext  // 使用缓存
    }

    // 生成新证书
    val expiry = Date(System.currentTimeMillis() + HOST_CERT_VALIDITY_MS)
    val sslContext = generateHostCertificate(host, expiry)

    // 缓存 24 小时
    certCache[host] = CachedCert(
        sslContext = sslContext,
        expires = System.currentTimeMillis() + HOST_CERT_VALIDITY_MS
    )

    return sslContext
}

data class CachedCert(
    val sslContext: SSLContext,
    val expires: Long
) {
    val isExpired: Boolean get() = System.currentTimeMillis() > expires
}
```

#### 优化建议实现

**1. 连接池（预计工时：6-8h）**
```kotlin
class ConnectionPool {
    private val pool = ConcurrentHashMap<String, Queue<SocketChannel>>()
    private val maxPoolSize = 50

    fun acquireConnection(host: String, port: Int): SocketChannel {
        val key = "$host:$port"
        return pool[key]?.poll() ?: createNewConnection(host, port)
    }

    fun returnConnection(host: String, port: Int, channel: SocketChannel) {
        val queue = pool.computeIfAbsent("$host:$port") { ConcurrentLinkedQueue() }
        if (queue.size < maxPoolSize) {
            queue.offer(channel)
        } else {
            channel.close()
        }
    }
}
```

**2. 内存监控（预计工时：4-6h）**
```kotlin
class MemoryMonitor {
    private val runtime = Runtime.getRuntime()
    private val maxMemory = runtime.maxMemory()

    fun shouldPruneConnections(): Boolean {
        val usedMemory = runtime.totalMemory() - runtime.freeMemory()
        return usedMemory.toDouble() / maxMemory > 0.8  // 80% 阈值
    }

    fun getMemoryUsagePercent(): Double {
        val usedMemory = runtime.totalMemory() - runtime.freeMemory()
        return usedMemory.toDouble() / maxMemory * 100
    }
}
```

## 五、关键决策点

### 决策 1: HTTPS 解密实现方案

**方案 A: Flutter 端实现（推荐）**
- 优点：跨平台复用，iOS 扩展成本低
- 缺点：Dart 端需要完整实现 SSL 层，技术难度高
- 工时：32-40h

**方案 B: Android 原生实现**
- 优点：可复用现有 Android 网络库，实现相对简单
- 缺点：iOS 需要重新实现，跨平台复用差
- 工时：24-32h

**推荐：方案 A** - 虽然初期投入高，但长期维护成本低，符合跨平台战略

### 决策 2: 证书存储方案

**方案 A: Android KeyStore（推荐）**
- 优点：硬件级安全，私钥不可导出
- 缺点：仅 Android 可用，iOS 需要 Keychain 方案
- 工时：6-8h

**方案 B: 加密文件存储**
- 优点：跨平台实现一致
- 缺点：安全性弱于 KeyStore
- 工时：4-6h

**推荐：方案 A** - 安全性第一，符合应用商店审核要求

## 六、总结

Spider Proxy 项目技术基础扎实，核心功能已实现可用。当前主要风险集中在：

1. **安全性** - CA 私钥存储、证书有效期（必须修复）
2. **功能完整性** - HTTPS 解密未完成（核心阻塞项）
3. **测试覆盖** - 单元测试、集成测试缺失（质量保证）
4. **性能优化** - 动态缓冲区、连接池（用户体验）

建议按 P0 → P1 → P2 优先级逐步推进，预计 8-10 周后可具备上架条件。
