### 1.2 审计发现

**优势**:
- 架构清晰，模块职责分明

## 三、代码质量审计

### 3.1 架构设计

**优点**:
```
✅ 清晰的分层架构
   UI Layer (widgets, pages)
      ↓
   State Management (redux/value_notifier)
      ↓
   Core Layer (proxy, mapper, script, debugger)
      ↓
   Platform Channel (Android/iOS)

✅ 单一职责原则
   - BodyMapper: 请求体替换
   - WebSocketProxy: WebSocket 抓包
   - ScriptEngineV2: 脚本执行
   - BreakpointDebugger: 断点调试

✅ 依赖注入
   - ProxyService 统一管理各模块
   - PlatformChannel 封装原生调用
```

**改进建议**:
```
⚠️ 模块间耦合
   - ScriptEngineV2 直接依赖 ProxyService
   - 建议：通过事件总线解耦

⚠️ 缺少接口抽象
   - BodyMapper、WebSocketProxy 无接口定义
   - 建议：定义接口便于测试和扩展
```

### 3.2 代码规范

**检查结果**:

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 命名规范 | ✅ 通过 | 类名 PascalCase，变量 camelCase |
| 类型安全 | ✅ 通过 | 完整类型标注，无 dynamic 滥用 |
| 空安全 | ✅ 通过 | 正确使用 ?、!、required |
| 错误处理 | ⚠️ 部分 | 部分异步操作缺少 try-catch |
| 日志规范 | ⚠️ 部分 | 调试 print 未统一封装 |
| 文档注释 | ⚠️ 部分 | 公共 API 有注释，内部方法缺少 |

### 3.3 性能问题

**已实现优化**:
- ✅ BodyCache 5 秒 TTL 缓存
- ✅ TCP 连接池复用
- ✅ 动态缓冲区 (16KB-1MB)
- ✅ ValueNotifier 增量 UI 更新

**待优化**:
- ⚠️ BodyMapper 规则匹配为线性搜索 O(n)
  - 建议：使用 HashMap 索引优化到 O(1)
- ⚠️ WebSocket 消息列表无上限
  - 建议：添加环形缓冲区，限制最大消息数
- ⚠️ ScriptEngineV2 每次执行创建新上下文
  - 建议：上下文池复用

### 3.4 安全隐患

**已实现安全措施**:
- ✅ dart_eval 沙盒执行
- ✅ Android KeyStore 私钥存储
- ✅ 证书 24 小时过期

**潜在风险**:
- ⚠️ Map Local 文件读取无路径校验
  - 风险：目录遍历攻击
  - 建议：限制在应用沙盒目录内
- ⚠️ Map Remote 无 URL 白名单
  - 风险：SSRF 攻击
  - 建议：添加 URL 校验规则
- ⚠️ 脚本执行超时为硬编码
  - 建议：用户可配置

## 七、优化建议

### 7.1 短期优化 (1-2 周)

**优先级 P0**:

1. **完善 JSON 工具函数**
   ```dart
   // lib/core/script/script_api.dart
   'jsonDecode': Func1((str) => jsonDecode(str)),
   'jsonEncode': Func1((obj) => jsonEncode(obj)),
   ```

2. **添加文件监听**
   ```dart
   // lib/core/mapper/body_cache.dart
   import 'package:watcher/watcher.dart';

   class BodyCache {
     final _watcher = Watcher();

     void _watchFile(String path) {
       _watcher.events.listen((event) {
         if (event.path == path) {
           _cache.remove(path); // 清除缓存
         }
       });
     }
   }
   ```

3. **添加路径校验**
   ```dart
   // lib/core/mapper/body_mapper.dart
   bool _isSafePath(String path) {
     final appDir = await getApplicationDocumentsDirectory();
     return path.startsWith(appDir.path);
   }
   ```

### 7.2 中期优化 (1-2 月)

**优先级 P1**:

1. **WebSocket 压缩扩展支持**
   - 实现 permessage-deflate 扩展
   - 添加压缩/解压逻辑
   - 更新 UI 显示压缩信息

2. **规则匹配性能优化**
   - 使用 HashMap 建立 URL 索引
   - 添加正则表达式预编译缓存
   - 实现规则优先级队列

3. **脚本调试功能**
   - 添加脚本执行日志
   - 实现错误堆栈展示
   - 添加性能分析工具

### 7.3 长期优化 (3-6 月)

**优先级 P2**:

1. **架构重构**
   - 引入依赖注入框架 (injectable/get_it)

   - 实现插件系统

2. **数据持久化**
   - 使用 Hive 存储历史会话
   - 实现数据自动清理策略
   - 添加数据导出/导入功能

3. **跨平台扩展**
   - macOS/Windows 桌面端
