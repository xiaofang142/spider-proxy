## 第一部分：项目概述

### 1.1 项目简介

Spider Proxy 是一个基于 Flutter + Android 原生的网络代理和抓包工具，具备以下核心能力：

- **流量捕获** - 通过 Android VpnService 创建 TUN 设备捕获所有网络流量
- **HTTPS 解密** - 中间人 (MITM) 技术解密 HTTPS 流量
- **代理转发** - SOCKS5 代理转发 TCP/UDP 流量
- **流量分析** - 实时显示上传/下载速度，请求/响应详情

### 1.2 技术栈

| 层级 | 技术 |

| UI 框架 | Flutter 3.24+ |

| 协程 | Kotlin Coroutines |
| 非阻塞 I/O | Java NIO Selector |

### 2.2 流量处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        出站流量 (设备→代理)                       │
├─────────────────────────────────────────────────────────────────┤
│  1. TUN 设备读取 IP 数据包                                         │
│     ↓                                                           │
│  2. VpnService.forwardTraffic() 解析协议                          │
│     ↓                                                           │
│  3. 根据协议分发                                                   │
│     - TCP (6) → TcpForwarder.handleTcpPacket()                  │
│     - UDP (17) → UdpForwarder.handleUdpPacket()                 │
│     ↓                                                           │
│  4. 转发器管理连接并发送到 SOCKS5 代理                              │
│     ↓                                                           │
│  5. 代理服务器处理请求                                            │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                        入站流量 (代理→设备)                        │
├─────────────────────────────────────────────────────────────────┤
│  1. 代理服务器返回响应                                            │
│     ↓                                                           │
│  2. TcpForwarder/UdpForwarder 读取响应                           │
│     ↓                                                           │
│  3. TunDeviceWriter.writeTcpResponse()/writeUdpResponse()       │
│     ↓                                                           │
│  4. 构建 IP 包头部 (计算校验和)                                    │
│     ↓                                                           │
│  5. 写入 TUN 设备                                                  │
│     ↓                                                           │
│  6. 系统接收数据包并路由到应用                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 IP 包构建和校验和

```kotlin
private fun buildIpPacket(...): ByteArray {
    val packet = ByteArray(totalLength)
    var pos = 0

    // IP 头部
    packet[pos++] = 0x45.toByte()  // IPv4, 5 * 4 = 20 bytes
    packet[pos++] = 0x00           // 服务类型
    // 总长度 (16 位)
    packet[pos++] = ((totalLength shr 8) and 0xFF).toByte()
    packet[pos++] = (totalLength and 0xFF).toByte()
    // ... 其他字段

    // 计算校验和
    val checksum = calculateChecksum(packet, 0, ipHeaderLength)
    packet[10] = ((checksum shr 8) and 0xFF).toByte()
    packet[11] = (checksum and 0xFF).toByte()

    return packet
}

private fun calculateChecksum(data: ByteArray, offset: Int, length: Int): Int {
    var sum = 0
    for (i in offset until offset + length step 2) {
        sum += if (i + 1 < length) {
            ((data[i].toInt() and 0xFF) shl 8) or (data[i + 1].toInt() and 0xFF)
        } else {
            (data[i].toInt() and 0xFF) shl 8
        }
    }
    // 折叠 32 位到 16 位
    while (sum shr 16 != 0) {
        sum = (sum and 0xFFFF) + (sum shr 16)
    }
    return sum.inv() and 0xFFFF
}
```
