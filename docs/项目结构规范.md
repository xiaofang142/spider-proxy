## 📝 二、命名规范

### 文件和目录命名
- **目录**：小写 + 下划线（`snake_case`）
  - ✅ `core/`, `features/`, `traffic_analysis/`
  - ❌ `Core/`, `Features/`, `trafficAnalysis/`

- **Dart 文件**：小写 + 下划线（`snake_case`）
  - ✅ `http_proxy.dart`, `rule_engine.dart`
  - ❌ `HttpProxy.dart`, `ruleEngine.dart`

- **类名**：大驼峰（`PascalCase`）
  - ✅ `HttpProxy`, `RuleEngine`, `TrafficAnalyzer`
  - ❌ `httpProxy`, `rule_engine`

- **函数/变量**：小驼峰（`camelCase`）
  - ✅ `startProxy()`, `trafficData`, `filterRules`
  - ❌ `StartProxy()`, `traffic_data`

- **常量**：全大写 + 下划线（`UPPER_SNAKE_CASE`）
  - ✅ `MAX_CONNECTIONS`, `DEFAULT_PORT`
  - ❌ `MaxConnections`, `defaultPort`

## 🏗️ 四、模块职责

### core/ - 核心模块
- **职责**：提供底层基础设施，无业务逻辑
- **依赖**：仅依赖 Dart 核心库和基础第三方包
- **特点**：高内聚、低耦合、可独立测试

### features/ - 功能模块
- **职责**：实现具体业务功能
- **依赖**：可依赖 core/ 模块
- **特点**：按功能垂直划分，自包含

### ui/ - UI 模块
- **职责**：用户界面和交互
- **依赖**：可依赖 core/ 和 features/
- **特点**：遵循 Flutter 最佳实践，组件化

### 测试代码模板
```dart
import 'package:flutter_test/flutter_test.dart';
import 'package:spider_proxy/core/proxy/http_proxy.dart';

void main() {
  group('HttpProxy', () {
    late HttpProxy proxy;
    
    setUp(() {
      proxy = HttpProxy(port: 8080);
    });
    
    tearDown(() {
      proxy.dispose();
    });
    
    test('should start successfully', () async {
      await proxy.start();
      expect(proxy.isRunning, isTrue);
    });
    
    test('should stop successfully', () async {
      await proxy.start();
      await proxy.stop();
      expect(proxy.isRunning, isFalse);
    });
  });
}
```

## 🔄 七、Git 工作流

### 分支命名
- `main` - 主分支
- `develop` - 开发分支
- `feature/<功能名>` - 功能分支
  - ✅ `feature/proxy-engine`
  - ✅ `feature/ui-homepage`
- `bugfix/<问题描述>` - 修复分支
- `release/<版本号>` - 发布分支

### Commit 规范
```bash
# 格式：<type>(<scope>): <subject>

# 类型
feat:     新功能
fix:      修复 bug
docs:     文档更新
style:    代码格式（不影响功能）
```
